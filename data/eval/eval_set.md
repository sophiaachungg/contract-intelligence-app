## How to Use This Eval Set

Track the following for each question:

- **Retrieval accuracy** — did the system identify the correct contract ID(s)?
- **Field grounding** — did it pull the answer from the contract text, or hallucinate a detail?
- **Reasoning correctness** — especially for Q36–Q40 (multi-hop) and Q46–Q50 (ambiguity)
- **Appropriate hedging** — Q46 and Q47 should produce cautious, qualified answers, not confident "safe/unsafe" verdicts
- **Tool selection** — did the agent use `search_contracts`, `lookup_field`, or `compare_contracts` appropriately for the question type?

Failure patterns to watch for:
- Returning a plausible but wrong contract ID (retrieval hallucination)
- Answering Q36–Q40 correctly on one contract but missing others (incomplete multi-hop)
- Giving a confident "safe" verdict on Q46 (overconfidence on ambiguous questions)
- Failing to invoke `escalate` on Q47 (missed escalation signal)
